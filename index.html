01 / ANPR
🚗
Automatic Number Plate Recognition System

Not a straight line from idea to product — a months-long grind through broken OCR engines, exploding false positives, a dataset that didn't exist, and a live-camera that refused to cooperate. Every obstacle forced a smarter solution. The result: a stable, production-deployed ANPR system that actually works.

▶ The Full Journey
Phase 1 — OCR Engine Graveyard. The search for a working OCR engine started broad. Tried nearly every available option — none delivered satisfying accuracy on Indian number plates. Eventually settled on YOLOv8m + PaddleOCR as the baseline. But it was painfully slow, missed characters regularly, and was completely unstable on a live camera feed. Added Indian plate regex post-processing — this improved results noticeably, but couldn't fully fix the accuracy gap.
Tesseract ✗ EasyOCR ✗ PaddleOCR (raw) ✗ PaddleOCR + regex → partial ✓
Phase 2 — False Positive Crisis. Any rectangular object in frame — storefronts, stickers, camera watermarks, signboards — was being confidently detected as a number plate. Worse: front-facing plates weren't being detected at all, only rear plates. Collected and manually annotated front-plate images to fill the detection gap. That fixed detection, but false positives still ran wild.
Storefront → plate ✗ Watermark → plate ✗ Front plates missed ✗ Front plate annotations → detection fixed
Phase 3 — The Dataset That Didn't Exist. To fix OCR accuracy properly, the plan was to build a lightweight custom recognition model. Problem: couldn't find any dataset with actual plate text labels for recognition training. Tried every available source — public repos, scraping, synthetic generation. Nothing worked. So the only option was to do it manually.

681 plate images annotated by hand over 12 straight hours. Then extended programmatically to 2,500 images. Later discovered a public 1-lakh character recognition dataset — combined all characters with the manual annotations. After full quality filtering (deduplication, blur scoring, noise removal), ended up with 14K images for detection and 17K images for recognition.
681 manual annotations 12 hours of labeling 2,500 → 17K via pipeline 14K detection · 17K recognition
Phase 4 — TrOCR Breakthrough, Then Another Wall. Fine-tuned Microsoft TrOCR on the curated dataset — it worked well on static images, hitting strong character accuracy. But on a live feed with moving vehicles, it was still unstable. Came very close to solving this, but moving-vehicle inference remained unresolved despite exhausting every approach. Fell back to PaddleOCR, which still failed on edge cases. Added a manual entry fallback as a practical, user-facing workaround.
TrOCR → static images ✓ TrOCR → moving vehicles ✗ PaddleOCR fallback + manual entry
Phase 5 — False Positives, Solved for Good. Researched hard negative mining. Collected 10,000 negative images — storefronts, signboards, stickers, watermarks, anything that looked plate-shaped but wasn't. Ran training in two rounds: first on the 14K plate dataset alone, then again on 14K plates + 10K negatives combined. This completely eliminated the false positive problem.

The final ANPR system is stable under real-world conditions, accurately handles diverse Indian plate formats, and has been running in a production environment on AWS EC2 via FastAPI.
10K hard negatives collected Two-round progressive fine-tuning False positives eliminated ✓ Production deployed ✓

03 / BIOMETRIC
🛡️
Secure Face Recognition with Anti-Spoofing

A full month spent testing every available face recognition approach before finding what actually works. Then building an anti-spoofing layer from scratch because no off-the-shelf solution was good enough. The outcome: a 3-stage production pipeline that handles real-world identity verification.

▶ The Full Journey
Phase 1 — One Month in the Library Graveyard. Spent a full month systematically testing every mainstream face recognition library. DeepFace, OpenCV face modules, and the face_recognition library — each tried, evaluated, and discarded. Performance on varied lighting, different angles, and partial occlusion was too inconsistent for production use. None cleared the accuracy bar required for a real deployment.
DeepFace ✗ face_recognition ✗ OpenCV face module ✗
Phase 2 — Building a Siamese Network from Scratch. Decided to go custom. Designed and trained a Siamese network with contrastive loss — learning to push similar faces together and dissimilar faces apart. Built OpenCV-based data collection scripts to capture personal training data. Rewrote it with triplet loss (anchor / positive / negative pairs) for better metric learning. Results were promising and performance improved — but still fell short of production-grade accuracy.
Siamese + Contrastive Loss ✗ Siamese + Triplet Loss ✗ Custom data collection pipeline ✓
Phase 3 — FaceNet Breakthrough. Discovered facenet_pytorch — InceptionResnetV1 pretrained on the VGGFace2 dataset. Cosine similarity for 1:N identity verification. The accuracy jump was immediate and significant — fast enrollment, reliable matching across lighting and angle variation. This became the identity verification backbone of the entire system.
FaceNet (InceptionResnetV1) ✓ Cosine similarity 1:N ✓ VGGFace2 pretrained backbone ✓
Phase 4 — Anti-Spoofing: Built from Scratch. The system could now recognize faces accurately — but it couldn't tell the difference between a real person and a printed photo, a passport-size image, or a face displayed on a phone screen. Any physical attack would pass straight through.

No existing off-the-shelf anti-spoof model met the requirements. Collected 10,000 images across 5 people — real faces under natural lighting, attacks via digital screens, and printed passport-size photos. Trained an EfficientNet-B0 binary classifier (live vs. spoof) on this custom dataset. The classifier was then integrated with YOLOv11-face for detection into a clean 3-stage pipeline: detect face → check liveness → verify identity. Deployed the full system on AWS EC2 via FastAPI.
10K custom anti-spoof images 5 people · 3 attack types EfficientNet-B0 liveness classifier ✓ 3-stage pipeline deployed ✓
